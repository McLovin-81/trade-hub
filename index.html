Legend: Explanation of HTML Tags Used

  • <!DOCTYPE html>: Declares the document type and version of HTML being used (HTML5 in this case).
  • <html>: The root element that wraps all content of the web page.
  • <head>: Contains metadata, links to stylesheets, and the page title. It's not visible on the page itself.
  • <meta charset="UTF-8">: Specifies the character encoding for the document, ensuring correct display of text.
  • <meta name="viewport" content="width=device-width, initial-scale=1.0">: Ensures the page is responsive, scaling properly on different devices by setting the viewport width and initial zoom level.
  • <title>: Sets the title of the web page, which appears in the browser tab.
  • <link rel="stylesheet" href="style-sheets/styles.css">: Links an external CSS stylesheet for styling the page.
  • <header>: Represents a group of introductory or navigational aids, such as the site's header.
  • <nav>: Represents a section of the page intended for navigation links, such as the main navigation bar.
  • <a>: Defines a hyperlink, which can be used to link to other web pages or resources. It can also wrap elements to make them clickable (like the logo in this case).
  • <img>: Embeds an image into the page. The alt attribute provides alternative text for screen readers or when the image fails to load.
  • <div>: A generic container element used to group content, primarily for styling and layout purposes.
  • <button>: Represents a clickable button used for various actions (e.g., form submission, toggling features).
  • <i>: Used for icons or italicized text. Here, it embeds a Font Awesome icon.
  • <script>: Links to an external JavaScript file that adds interactivity or other functionality to the page.
  • <footer>: Represents the footer of the page, typically containing information about the document or site, like copyright details or additional navigation links. In this case, it's used to house the legend that explains the HTML tags and attributes used in the document.
  • <section>: Represents a standalone section of content within the document, usually with its own heading. Here, it is used to group the legend content within the footer.
  • <h2>: Defines a second-level heading, used to introduce the legend section. Headings help structure content hierarchically.
  • <ul>: Defines an unordered list, which is a collection of list items (<li>) presented with bullet points.
  • <li>: Represents a list item within an ordered (<ol>) or unordered (<ul>) list. Each tag or concept explained in the legend is a separate list item.
  • <p>: Defines a paragraph of text. In this case, it introduces the section about ARIA attributes, providing a brief description before listing them.

ARIA (Accessible Rich Internet Applications) attributes:

  • aria-label: Provides an accessible label for an element, typically used by screen readers. It helps describe the function or purpose of an element when there isn't visible text that provides this information (e.g., an icon button).
  • aria-haspopup: Indicates that the element has a popup menu or dialog associated with it.
  • aria-expanded: Used to indicate whether a collapsible element (like a dropdown menu) is currently expanded or collapsed.
